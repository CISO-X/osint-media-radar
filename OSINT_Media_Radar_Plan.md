# OSINT Media Radar — План реализации MVP

## Концепция

**OSINT Media Radar** — веб-приложение для мониторинга и анализа информационного фона вокруг компаний и персон. Аналитик вводит ключевые слова (название компании, ФИО, тему) — приложение собирает новости из открытых источников, анализирует тональность, строит таймлайн упоминаний и генерирует краткую аналитическую сводку.

## Стек технологий

| Компонент | Технология | Почему |
|-----------|-----------|--------|
| Backend | **Python 3.11 + FastAPI** | Быстрый API, богатая экосистема NLP-библиотек, асинхронность |
| Frontend | **React + Tailwind CSS + Recharts** | Современный UI, красивые графики из коробки |
| База данных | **SQLite** | Для MVP достаточно, zero-config, файловая БД |
| NLP / Тональность | **dostoevsky** (русский) + **TextBlob** (английский) | Лучшие бесплатные решения для сентимент-анализа |
| NER (извлечение сущностей) | **Natasha** (русский) | Извлечение имён, компаний, мест из русскоязычных текстов |
| Источники новостей | **RSS-парсер (feedparser)** + **GNews API** + **NewsAPI** (free tier) | Бесплатные и легальные источники |
| PDF-экспорт | **WeasyPrint** или **ReportLab** | Генерация красивых PDF-отчётов |

## Функционал MVP (v1.0)

### Модуль 1: Поиск и сбор новостей
- Поле ввода: ключевые слова, название компании или ФИО
- Фильтры: период (последняя неделя / месяц / год), язык (RU/EN)
- Источники: RSS-ленты крупных СМИ (ТАСС, РБК, Коммерсант, Ведомости, vc.ru) + GNews API
- Дедупликация похожих статей

### Модуль 2: Таймлайн упоминаний
- Интерактивная временная шкала: ось X — даты, ось Y — количество упоминаний
- Цветовая кодировка по тональности (зелёный / жёлтый / красный)
- Клик по точке — раскрытие списка статей за эту дату

### Модуль 3: Анализ тональности
- Автоматическая классификация каждой статьи: позитивная / нейтральная / негативная
- Круговая диаграмма общего распределения
- Гистограмма тональности по времени (тренд настроений)

### Модуль 4: Извлечение сущностей (NER)
- Автоматическое выделение из текстов: персоны, организации, локации
- Облако связанных сущностей (кто/что ещё упоминается вместе с объектом поиска)
- Таблица топ-10 связанных сущностей с частотой упоминания

### Модуль 5: Executive Summary
- Автоматическая генерация краткой сводки (3-5 предложений) по результатам анализа
- Ключевые факты: сколько упоминаний, тренд тональности, главные темы
- Красные флаги: резкий рост негатива, скандалы, судебные упоминания

### Модуль 6: Экспорт
- PDF-отчёт с графиками, таймлайном и executive summary
- CSV-экспорт списка статей для дальнейшего анализа

## API Endpoints

| Метод | Путь | Описание |
|-------|------|----------|
| `POST` | `/api/search` | Запустить поиск по ключевым словам |
| `GET` | `/api/results/{search_id}` | Получить результаты поиска |
| `GET` | `/api/timeline/{search_id}` | Данные для таймлайна |
| `GET` | `/api/sentiment/{search_id}` | Данные анализа тональности |
| `GET` | `/api/entities/{search_id}` | Извлечённые сущности |
| `GET` | `/api/summary/{search_id}` | Executive summary |
| `GET` | `/api/export/pdf/{search_id}` | Скачать PDF-отчёт |
| `GET` | `/api/export/csv/{search_id}` | Скачать CSV |
| `GET` | `/api/history` | История запросов |

## Бесплатные API и ключи

|--------|--------------------------|-------------|
| **GNews API** | 100 запросов/день | gnews.io |

## Правовые аспекты (по материалам курса)

Приложение работает исключительно с **открытыми источниками**:
- RSS-ленты — публичные, предназначены для машинного чтения
- Новостные API — официальные интерфейсы с лицензионными условиями
- Никакого парсинга закрытых разделов, авторизованного контента или персональных данных
- Соответствие принципам OSINT: легальность, этичность, методичность
